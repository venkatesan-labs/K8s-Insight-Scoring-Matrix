Namespace

 

Namespace

  • Logical separation of cluster resources within the same physical cluster.
  • Resource allocation, management, and quota application at the namespace level.
  • Access control and security policies to be applied per namespace.
  • Simplified monitoring and organization of related resources by grouping them under a namespace name.

 

# Imperative command to create namespcae

# kubectl create namespace my-namespace

apiVersion: v1

kind: Namespace

metadata:

  name: my-app

# Labels in namespace used to filter the namespace

# kubectl get pods -n (kubectl get namespace -l environment=production) >> get pods in particular name space)

# kubectl get namespaece -l environment=devops 

  labels:

    app: my-app

# annotation will provide an detailed description and information of this namespace yaml

  annotations:

    description: This is default namespace for entire template

# when object (pod or deployment) under the namespace is removed, it will make sure all the Object in namespace are removed and finally remove this namespace.

  finalizers:

    - kubernetes
